# AM-Line
Experimental messenger without message history; only live typing fields for participants.

# Спецификация идеи: AM-Line 

## Концепция

Приложение похоже не на классический мессенджер с историей, а на "комнату реального времени", где:

- Есть комната и список участников.
- У каждого участника есть одно текстовое поле, которое он редактирует.
- Остальные участники в реальном времени видят текст, который сейчас печатает человек.
- Нет кнопки "Отправить сообщение" и нет ленты сообщений. Есть только текущее состояние фразы каждого участника.

Цель: создать формат общения, который ближе к устному разговору:
- Ты говоришь — и это существует только "здесь и сейчас".
- Нет долговременной истории, пересылок, цитат.

## Возможные сценарии использования

1. Честные разговоры / терапия / коучинг
2. Приватные обсуждения без сохранения истории
3. Экспериментальные форматы общения, арт-проекты, брейнштормы

## Техническая модель (общая)

- Клиент подключается по WebSocket (или WebRTC data channel).
- Для каждой комнаты сервер хранит только:
  - список подключённых пользователей,
  - текущее "поле текста" для каждого пользователя (в оперативной памяти).
- При изменении текста клиент посылает событие `typing`:
  - `{ user_id, room_id, text }`
- Сервер рассылает это событие другим участникам комнаты.
- При выходе всех участников данные комнаты удаляются (ничего не сохраняется в БД).

## Отличия от обычных мессенджеров

- Нет истории сообщений.
- Нет списков "кто что когда отправил".
- Упор на эфемерность и эффект "живого эфира".
