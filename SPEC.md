# AM-Line — Anti Messenger (SPEC)

Автор: Aleksei Drobin  
Дата: 2025-12-01  

## 1. Идея

AM-Line — это текстовый чат **без истории сообщений**.

- У каждого участника есть **своё поле ввода**.
- Другие видят, **что он печатает в реальном времени**.
- Нет ленты сообщений и кнопки «отправить».
- После окончания сессии данные **не сохраняются**.

По смыслу это ближе к устному разговору:  
сказал — и оно существует только в момент диалога.

---

## 2. Как это работает (упрощённо)

- Есть **комната** (`room`), в ней несколько участников.
- Каждый участник подключается к комнате.
- Клиент отправляет на сервер текущее содержимое поля (событие `typing`).
- Сервер рассылает этот текст другим участникам комнаты.
- Пока кто-то в комнате есть, хранится:
  - список участников,
  - их текущий текст.
- Когда все вышли из комнаты, данные о комнате удаляются.

---

## 3. Для кого и зачем

Примеры использования:

- приватные разговоры без сохранения истории;
- “честные разговоры” / терапия / коучинг;
- экспериментальные форматы общения, поток сознания, арт-проекты.

---

## 4. Минимальный функционал (MVP)

1. Создать комнату и получить ссылку / `room_id`.
2. Подключиться к комнате по ссылке.
3. Ввести никнейм.
4. Видеть список участников.
5. Видеть “живые” поля текста других участников.
6. При изменении текста у себя — обновление у всех остальных.

---

## 5. Приватность

- Не хранить историю сообщений в базе данных.
- Не логировать содержимое текста в файлы логов.
- Данные о комнатах и текстах жить только в памяти сервера.
- Для реального использования — шифрованное соединение (`wss://`).
